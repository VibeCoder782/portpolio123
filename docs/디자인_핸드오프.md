# 포트폴리오 리뉴얼 — 클로드 디자인 핸드오프 브리프

> 작성: 2026-07-15, Claude Code 세션에서 정리. 이 문서 하나로 맥락 전체를 전달한다.
> 목표: 이 브리프로 클로드 디자인에서 룩을 확정하고, 결과물(코드 or 캡처)을 다시 Claude Code로 가져와 실제 저장소에 이식한다.

---

## 1. 프로젝트 개요

- **주인공**: 양순민 — PM/PO, 부산. 이직용 포트폴리오 사이트 리뉴얼
- **현재 사이트**: React 18 + Vite + TS 단일 페이지, Vercel 배포 (portpolio123-lemon.vercel.app)
- **한 줄 정체성**: "기획만 하지 않고 직접 만들어 증명하는 실행형 PO"
- **방문자**: 채용담당자 (체류 1~3분, 정보를 빨리 읽을 수 있어야 함)

## 2. 핵심 서사 (사이트가 들려줄 이야기)

```
12년을 현장에서 (도미노피자 매장 매니저, 2009–2021)
→ 34세, 방법을 바꿨다 (웹 기획자로 전향, 2021)
→ 3년의 프로덕트 (아리모아 사원→대리→과장 / 핸디 개발1팀 이사·PO)
→ 이제는 직접 만든다 (AI·자동화로 10+개 제품을 직접 설계·개발)
```

- **태그라인**: "안 되는 건 없다, 방법이 다를 뿐."
- **히어로 카피(확정)**: 영문 대형 3행 `12 YEARS ON THE FLOOR, / 3 IN PRODUCT. / NOW — I BUILD.` + 한글 소형 병기
- **키 넘버**: 12년 · 3년 · 60개 사이트/연기 0건 · 프로젝트 18건 · 사례 6건 · 빌드 10+

## 3. 확정된 취향 (사용자가 직접 고른 것)

- **블랙 & 화이트** 톤. 색은 포인트로만
- **샤프한 초대형 타이포**가 주인공 (영문 디스플레이 — 목업은 Anton 사용, 국문 Pretendard)
- **유리(글라스)·입체 요소는 "포인트"** — 주인공 아님, 반드시 의미가 있어야 함
- 3D 입체감/공간감을 원하지만 **콘텐츠가 곧 무대**여야 함 (배경이 따로 놀면 실패)
- **콘텐츠 즉시 확인 가능** — 숨겨놓고 찾게 하지 않기
- **부담 없는 절제** — 상시 움직이는 효과 금지, 움직임은 스크롤 등장(1회성)과 호버(사용자 의도)만
- 사용자가 좋아하는 순간들: ①카메라가 빨려 들어가는 물리감 ②대형 타이포가 장면처럼 등장 ③마우스 인터랙션 디테일 ④스크롤마다 느껴지는 입체감

**레퍼런스 위계**:
- 축: **rayraylab.com** (서울 1인 스튜디오 — "3차원 요소를 2D 웹에 넣어 깊이감" 철학)
- 상위 취향: Active Theory, Lusion (단, 이들의 핵심은 파티클이 아니라 **콘텐츠가 WebGL 안에 통합**돼 있다는 점)
- 절제의 기준: Minh Pham ("3D는 콘텐츠를 받치는 보조")
- 참고: taotajima.jp (콘텐츠 즉시 보이는 3D 전환), joseph-san.com (스크롤 3D 월드)

## 4. 실패의 역사 — 하지 말 것 (전부 실제로 반려된 것들)

1. **파티클 배경 씬** (볼류메트릭 파티클 + 카메라 저니 구현까지 갔다가 폐기)
   → "콘텐츠랑 배경이랑 따로 논다". 어떤 포트폴리오에도 붙는 범용 효과는 금지
2. **의미 없는 장식 오브젝트** (회전하는 유리 프리즘) → "뜬금없다". 3D/유리는 서사적 의미를 설명할 수 있을 때만
3. **기존 구조 재활용** — 세로 섹션 스택 + 중앙 히어로/태그라인 + "Chapter XX" 라벨 + 카드 그리드를 기본값으로 쓰는 것 → "옛날 디자인을 못 벗어난다"
4. **과한 절제로 문서화** — 중앙 정렬 컨테이너에 표·리스트만 나열 → "표현방법이 너무 별로"
5. **표 형태의 아카이브 리스트** → 반려. 숨겼다 드러나는 표현(크레딧 스타일)으로 대체함
6. 몰입형 게임(아바타 조작)형 → "과하고 내 컨셉과 안 맞음"
7. **한 화면에 한 섹션** 원칙 — 타이포가 뷰포트를 넘쳐 흐르면 안 됨 (vw뿐 아니라 vh에도 맞출 것)

## 5. 현재 도달점 — Mock A v3 "포스터의 연속" (이것이 출발점)

사용자가 승인한 마지막 목업 (`public/mock-a2.html`, 로컬). 특징:

- **씬 8개, 흑백 완전 교차**: 히어로(백)→전향(흑)→성장(백)→빌즈(흑)→아카이브(백)→사례(흑)→스킬(백)→콘택트(흑). 반전 자체가 서사(백=종이/과거, 흑=화면/현재)
- 반전 순간 은은한 **유리 스윕** 컷
- **기념비 숫자**: 60·18·6이 계단 그림자를 단 초대형 Anton 타이포로 장면에 서 있음
- **장면마다 구도가 다름** (중앙 정렬 반복 금지): 좌하단 앵커 / 좌 숫자·우 콘텐츠 / 거울 반전 / 3단 비대칭
- **Builds = 영문 대문자 타이포 원장** (AI AUTOMATION / BOOGION / MOUNTAINON / BOOKITDA / AI INSIGHT OS + View All): 호버하면 그 줄 뒤로 **컬러 아트 타일**(입체 구체 그라데이션) 또는 스크린샷이 떠오름 — 흑백 세계에서 유일한 컬러는 호버했을 때만
- **아카이브 18건 = 엔딩 크레딧**: 스크롤 진입 시 한 줄씩 떠오르고, 평소 42% 흐림, 호버 시 선명
- **사례 6건 = 보고서 미학** (Rep 01~06, 문제→결과)
- **Contact = 유리 명함** (글라스 카드에 이름·이메일) — 유리가 의미를 갖는 유일한 곳
- 히어로 등장: 줄 단위 클립 리빌 1회. "I build."는 아웃라인→호버 시 채움
