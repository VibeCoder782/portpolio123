## 0. 디자인 제1원칙 — "면이 아니라 공간"

모든 장면은 **z축으로 깊은 볼류메트릭 파티클 필드** 안에서 벌어진다.
카메라는 그림을 바라보지 않고 **공간을 통과**한다. 입체감의 4대 장치:

| 장치 | 효과 |
|---|---|
| **시차 속도차** | 가까운 파티클은 빠르게 스쳐 지나가고 먼 것은 느리게 — 빨려드는 물리감의 정체 |
| **근경 더스트 레이어** | 카메라 바로 앞을 스치는 큰 흐릿한 입자 소수 (draw call +1, 총 2개 허용) |
| **깊이 안개(fog) + 크기 감쇠** | 멀수록 어둡고 작게 — 공간의 끝이 안 보이게 |
| **DOM 깊이 스테이징** | 타이포·카드도 스크롤 시차 레이어(CSS perspective)로 z축에 배치 |

마우스: 카메라 리그에 **마우스 오비트 오프셋**(lerp) — 화면 전체가 시선을 따라 미세하게 회전 (모든 장면 공통). 커스텀 커서 + 자석 버튼 유지.

타이포: 내레이션은 **타이틀 카드 연출 언어** 하나로 통일 — 깊은 곳에서 스케일업하며 등장(z축 접근감) → 자간 정착 → 부드럽게 이탈. 장면처럼 등장하고 장면처럼 퇴장.

## 1. 시나리오 — 씬 바이 씬 (볼류메트릭 개정)

### SCENE 0 — 오프닝 (로더)
- 000→100%. 어둠 속 **3D 불씨 구름 내부에** 카메라가 이미 들어와 있음. 주위에서 불씨가 하나둘 점화 — 사방에 깊이감
- 100% → 타이틀 인

### SCENE 1 — 타이틀 「양순민」 (0~10%)
- 카메라가 불씨 구름 한가운데 정지. 마우스를 움직이면 **구름 전체가 시차 회전** — 첫 화면부터 "여긴 3D 공간"임을 각인
- 대형 타이포: 이름 + "안 되는 건 없다. 방법이 다를 뿐." (근경 불씨가 타이포 앞을 가끔 스침 — 깊이의 증거)

### SCENE 2 — 전향 (핀 #1 · 10~20%) ★빨려듦 #1
- 스크롤 = 카메라가 **전진 가속**. 불씨들이 좌우로 갈라지며 스쳐 지나감 (워프 직전)
- 중반: 불씨 일제 비산 → 전방 깊은 곳에 **차가운 블루 3D 격자(래티스)** 재결정화
- 카메라가 격자의 **회랑(corridor) 속으로 진입** — 격자 기둥들이 좌우·상하로 스쳐 지나가는 통과감
- 내레이션: "12년의 현장을 뒤로하고" → "34세, 방향을 바꿨다" (깊은 곳에서 접근하며 등장)

### SCENE 3 — 성장의 회랑 (About+Stats+Experience · 20~45%) ← 최대 개정
- **3D 터널/회랑 구조**: 60개의 웹사이트 창(패널)이 회랑 양옆·위아래에 z축으로 늘어서 있고, 카메라가 그 사이를 **전진 트래킹**
- 스쳐 지나가는 창들이 카운터와 동기 (0→60개 사이트 · 연기 0건 · 18 프로젝트)
- 사원→대리→과장→이사 타임라인 마일스톤이 회랑 벽면처럼 z축 순서로 배치 — **스크롤 = 시간을 통과**
- About/Experience 텍스트 카드는 DOM 시차 레이어로 좌우에서 다른 속도로 흐름
- 색: 블루 → 에메랄드

### SCENE 4 — 새벽의 줄 (Case Studies · 45~60%)
- 카메라 하강, **지면(ground plane)** 등장 — 파티클 대기줄이 z축 저 멀리까지 소실점으로 이어짐 (원근의 줄)
- 스크럽: 줄 선 점들이 떠올라 카메라 옆을 지나 **뒤로 흘러가는 스트림** — 새벽 네이비 → 아침빛
- 사례 카드 6건 (아르피나 센터피스). 인라인 스크럽, 핀 없음
- 내레이션: "문제는 현장에서 풀었다"

### SCENE 5 — 직접 만든다 (Builds · 핀 #2 · 60~82%) ★빨려듦 #2
- 파티클이 **3D 노드 성좌**로 — 노드들이 서로 다른 깊이에 떠 있고 엣지 펄스가 공간을 가로지름
- 카메라 **궤도 회전 + 서서히 진입** — 시스템 내부를 둘러보는 시점
- 스크럽: 노드 연결마다 프로젝트 카드(10장) 안착. 카드도 깊이 순서로 등장 (가까운 것 먼저)
- 내레이션: "이제, 직접 만든다"

### SCENE 6 — 수렴 (Skills+Contact · 82~100%)
- 성좌가 풀리며 전 파티클이 **전방 한 점**으로 수렴 — 카메라 천천히 풀백, 점이 YSM. 로고 광원이 됨
- 내레이션: "다음 이야기를 함께" + Contact. 엔딩 크레딧 푸터

## 2. 기술 구조 (깊이 우선 개정)

```
[Director]  GSAP ScrollTrigger 마스터 타임라인
     ├─→ [ParticleEngine]  볼류메트릭 필드 + 포메이션 모프 + fog/크기감쇠 (draw call 1)
     ├─→ [DustLayer]       근경 대형 입자 소수 (draw call 1)
     ├─→ [CameraRig]       3D 경로 키프레임 + 마우스 오비트 오프셋(lerp)
     └─→ [DOM Layer]       타이틀 카드 타이포 + 시차 섹션 (CSS perspective/translateZ)
```

- 포메이션 좌표는 전부 **3D 부피로 생성** (사전 계산, attribute 업로드 1회)
- 셰이더: 인접 포메이션 mix(per-particle 노이즈 지연) + fog + 거리 크기감쇠 + 근접 페이드(카메라 바로 앞 입자는 커지며 투명해짐 — DOF 대용)
- 카메라 경로: Catmull-Rom 3D 스플라인. 핀 구간은 ScrollTrigger 제어
- 성능 예산: draw call ≤2, 데스크톱 15k+더스트 / 모바일 9k, 60/30fps+, 번들 +80KB
- reduced-motion: 카메라 고정 + 챕터 정적 포스터 + 콘텐츠 즉시 표시. fps<40 저사양 폴백

### 파일 구조

```
src/journey/
  config.ts        ← 챕터·카메라 키프레임·색·내레이션 단일 소스

  CameraRig.ts     formations/ (F0~F6, 전부 3D 볼륨 생성기)
src/components/

  sections/ (시차 레이어 적용)
```
