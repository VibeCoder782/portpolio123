## 0. 목표와 원칙

**목표 한 줄:** 효과 백화점을 걷어내고, *하나의 파티클 씬*이 페이지 전체를 관통하며 **스크롤이 카메라를 씬 속으로 밀어넣는** 몰입형 포트폴리오로 재구조화한다.

**근거 (레퍼런스 재분석):**
- Pacôme Pertant — 시그니처 효과는 3~4개뿐(마우스 트레일, 나선↔리스트 갤러리, 푸터/페이지 전환). 전부 하나의 모션 언어로 통일. GSAP + Three.js.
- Minh Pham — "3D는 콘텐츠를 받치는 보조". 타이포 우선, 자신감 있는 절제.
- "빨려들어감"의 정체 = **핀(pin) + 스크럽(scrub) + 카메라 돌리(dolly-in) + 지속되는 단일 씬**. 효과 개수가 아니라 스크롤 구조의 문제.

**모션 시스템 (전 단계 공통 규칙):**
| 항목 | 규칙 |
|---|---|
| 이징 | `cubic-bezier(.16,1,.3,1)` **단 하나** (기존 fade류와 동일 계열) |
| 시간 | 0.3s(마이크로) / 0.6s(요소) / 1.0s(장면) — 3단만 사용 |
| 색 | 에메랄드(#10B981)는 **인터랙티브 요소에만**. 장식 남용 금지 |
| 스크롤 | 등장 애니메이션은 "지나가기"(fade-up) 최소화, 핵심 순간은 "조종하기"(scrub) |
| 접근성 | `prefers-reduced-motion` 시: 씬 정지·카메라 고정·핀 해제·즉시 표시 |

## Phase 0 — 빼기 (Subtraction)

> 목적: "난잡함"의 절반은 여기서 해결. 위험도 낮음, 즉시 체감.

### 제거 목록
| 항목 | 현재 | 조치 |
|---|---|---|
| 마우스 글로우 div | 커서 따라다니는 radial 300px | **삭제** (커스텀 커서와 역할 중복) |
| 스크롤 속도 스큐(skewY) | Builds 그리드가 기울어짐 | **삭제** (카메라 저니와 문법 충돌) |
| 둥둥 뜨는 이모지 float | 아이콘들 무한 상하운동 | **삭제** (시선 분산) |
| fabPulse 무한 펄스 | 챗봇 버튼 주변 파동 | **삭제**, hover 시에만 미세 반응 |
| stat-number gradientMove | 숫자 배경 무한 흐름 | **삭제**, 정적 그라데이션으로 |
| 마퀴 자동 스크롤 | 20s 무한 루프 | **삭제 또는** 스크롤 연동(스크럽)으로 대체 — 체크포인트에서 결정 |
| 페이지 와이프 패널 | 네비 클릭 시 검은 패널 | **일단 유지**, Phase 3에서 카메라 전환으로 대체 후 삭제 |

### 정돈 목록
| 항목 | 조치 |
|---|---|
| 히어로 성운 | 노이즈 변위 1.7→**0.8** (형태가 '구'로 읽히게), 반경 축소(뷰포트의 ~55%), 파티클 22k→**15k** |
| 타이포 위계 | 이름·태그라인 주변 여백 확대, 비네트 유지 |
| 이징/시간 | 흩어진 transition 값 전부 모션 시스템 표로 통일 |
| 나선(Builds) | Phase 2에서 핀+스크럽으로 재구축 예정이므로 임시로 강도 50% 감쇠 |

**산출물:** 정리된 사이트 스크린샷 비교(전/후) · 빌드 통과 · 콘솔 0 에러
**체크포인트 ①:** 사용자 확인 — "차분해졌는가?" + 마퀴 삭제/스크럽 결정

## Phase 1 — 지속되는 씬 + 카메라 저니 (몰입의 본체)

> 목적: 히어로에서 죽던 3D를 페이지 전체의 배경 우주로 승격. 스크롤 = 카메라 전진.

### 구조 변경
```
현재: <section hero> 안에 canvas (히어로 전용)
변경: <div fixed inset-0 z-0> 캔버스 (페이지 전체 고정 배경)
      모든 섹션은 z-1 레이어로 그 위에 흐름
```

### 카메라 저니 설계 (스크롤 진행도 p = 0~1)
| 구간(챕터) | 스크롤 위치 | 파티클 상태 | 카메라 |
|---|---|---|---|
| Hero | 0~10% | 구체(응집) | z=15 정지, 마우스 시차 |
| 탈출 | 10~20% | 구가 **터지며 흩어짐** | **z축 전진 가속** ← 빨려드는 순간 #1 |
| About~Experience | 20~55% | 성긴 별밭(저밀도, 어두움) | 천천히 전진, 콘텐츠 방해 금지 |
| Builds | 55~75% | 별밭이 **소용돌이로 재응집** | 감속·주시 ← 순간 #2 |
| Skills~Contact | 75~100% | 다시 구체로 수렴(수미상관) | 정지, 미세 시차 |

- 구현: 파티클 셰이더에 `uProgress` 유니폼 추가 → 정점을 "구체 위치 ↔ 흩어진 위치 ↔ 소용돌이 위치" 3개 어트리뷰트 간 mix. 카메라 z는 Lenis 스크롤 진행도에 lerp.
- 색: 챕터별 색온도 미세 변화(에메랄드 → 딥블루 → 에메랄드). 급변 금지.
- **성능 가드:** 파티클 렌더는 항상 1개 캔버스/1개 draw call. 모바일 9k 유지. `document.hidden` 및 reduced-motion 시 정지. 저사양 감지(첫 60프레임 fps<40) 시 정적 이미지 폴백.

**산출물:** 스크롤 전 구간 시연 영상/스크린샷 · fps 계측(데스크톱 60 / 모바일 30+)
**체크포인트 ②:** "빨려드는 느낌"이 실제로 나는지 사용자 확인 (여기가 승부처)

## Phase 2 — 핀 순간 2개 (스크럽 챕터)

> 목적: 스크롤을 "재생 바"로 만드는 결정적 순간. **딱 2개만.**

### 도구 도입: GSAP ScrollTrigger
- 핀+스크럽 업계 표준. Lenis와 공식 연동 패턴 존재(`lenis.on('scroll', ScrollTrigger.update)`).
- 의존성 +1 (gsap ~70KB gzip 전이라 수용 가능. 코드 스플리팅 검토)

### 핀 #1 — 히어로 탈출 (Hero Exit)
- 히어로가 **화면에 고정(pin)**된 채 스크롤하면: 제목이 살짝 확대되며 페이드 → 파티클 구가 폭발·가속 → 핀 해제되며 Stats로 자연 낙하
- 길이: 스크롤 1뷰포트 분량. 스킵 가능해야 함(과도한 스크롤 하이재킹 금지)

### 핀 #2 — Builds 조립 (현 나선의 완성형)
- Builds 섹션 진입 시 핀 → 스크롤 스크럽으로 9개 카드가 소용돌이에서 **한 장씩 순서대로** 그리드에 안착 → 마지막 카드 안착 시 핀 해제
- 현재의 "지나가면서 대충 풀리는" 나선을 "스크롤이 직접 조종하는" 조립으로 격상. 배경 파티클 소용돌이(Phase 1)와 동기화

**산출물:** 두 핀 구간 시연 · 모바일에서 핀 길이 축소 확인 · reduced-motion 시 핀 미적용 확인
**체크포인트 ③:** 핀 길이/속도 체감 튜닝

## Phase 3 — 전환 통일 (와이프 → 카메라)

- 네비 클릭 시: 검은 패널 와이프 **삭제** → 카메라가 파티클 사이를 고속 돌파해 해당 챕터 지점으로 이동(FTL 점프 느낌). 스크롤 저니와 같은 언어.
- 실패 폴백: reduced-motion 또는 저사양 → 즉시 점프(현행 유지)
- 로더(000→100%)는 유지하되 완료 시 "구체가 조립되며 시작"으로 연결(로더→씬 연속성)

**산출물:** 전 페이지 흐름 최종 시연
**체크포인트 ④:** 최종 승인 → 배포(Vercel) 논의

| 검증 | 각 Phase마다: tsc·build·콘솔 0·스크린샷 → 마지막에 적대적 리뷰 1회 재실행 |
